新的征途——用Python刷Leetcode

发表于 2016-11-06   |   分类于 生活   |  

前言

距离2017年只有50多天了，时光飞逝，十年前不再是1996，而是2006。以前一直觉得自己还很年轻，现在想想，也是到了要好好照顾身体，好好陪伴家人的年纪。
入职技威时代也有四个月了，三个月试用期转正后，就开始担任Linux软件组的代理组长，作为一个刚毕业的菜鸟，一个组里最晚入职的新人而言，挑战还是很大的。一开始，很担心和同事沟通不力，导致工作安排不能正常进行，好在同事们对于我的工作还是非常支持的，第一个月就在跌跌撞撞中前行，只能说是平庸，无功无过。
看了公司历史遗留的代码之后，一开始是非常不解，因为代码有点混乱；之后慢慢熟悉了工作内容和工作流程，熟悉了产品的功能之后，又开始佩服前辈们，程序中也有许多巧妙地想法。另一方面，实现一个复杂的系统需要良好的程序架构，显然一开始是有这个想法存在的，可惜经手的人多了，想法不能很好的传承，导致越来越混乱。为了尽可能地优化公司的现有程序，我需要在技术上有更多的了解，除了C语言，还要在算法、数据结构乃至设计模式上，有所涉猎。也许这是一个很长的旅程，也是一段艰难的征途，我准备上路了，生活总是充满挑战，人生才会充满乐趣。

征途

我的目标是学习Python和刷Leetcode的题目。
以前对于Python的学习，每次都是浅尝辄止，都没有使用Python实战过任何项目。对于算法，单纯的理论学习不实践，就“纸上得来终觉浅”，如果一直使用C语言来实现算法，未免太过于耗费时间。而Python的程序逻辑和实际语言更为接近，可以更好地体现算法的思路。追求算法效率时，可以使用c语言来实现；而为了高效地学习算法和程序思维，个人以为python更为合适。
因此，在后面的这段时间中，我希望自己能够在学习python的同时，使用python程序来解决Leetcode上的算法题目。也许对于很多人来说，Leetcode就像是以前的古诗文一样，是为了倒背如流以便于应付科技公司的面试。在我看来，Leetcode上的题目虽然是固定的，但是每一个题目，都会有一些技术大牛提供一些新颖的解法，对于新手而言，去欣赏优秀的代码能够更快更好的学习，何乐而不为呢？
选择Leetcode的另一个原因就在于，题目内容短小，可以在上班前看一道题目，上下班路上或者工作午休时稍微思考，也许刚开始时，想一天也不能得到一个解法，或者说不能通过，但是不停的学习，总会有进步的。现在和学校的课程不一样就在于，没有期末考试，不用为了考试而去应付，可以真实地面对自己的内心。现在开始努力，最坏的结果不过是，大器晚成。

计划

在2017年到来之前，熟练掌握Python编程语言，能够独立编写一些小型项目，如Python网络爬虫等。
完成Leetcode30道题目，这个数量低于一天一道，但是高于一天两天一道，最理想目标是能够一天完成一道题目，以我目前的能力，是无法完成的。不能好高骛远，又要有点挑战性，故给自己设定的是总数30道题目。
完成题目之后，将python语言的解法上传Github，同时尽可能的写博客记录这道题目的难点，并对一些有特点的解决方案进行分析。
这些计划，对于时间的要求很高，求其上者得其中，求其中者得其下，加油吧，少年！在这个最适合努力的年纪，不要选择偷懒。
