Fizz Buzz

题目简述

Write a program that outputs the string representation of numbers from 1 to n.
But for multiples of three it should output “Fizz” instead of the number and for the multiples of five output “Buzz”. For numbers which are multiples of both three and five output “FizzBuzz”.

Python 程序

作为Leetcode之旅的第一道题目，是一个非常简单的程序。

总结

这一道题目虽然很简单，但是对于我而言，也不是快速就解决了的。主要的阻碍是，之前一直学习C语言，近期初学Python，对于Python语言的基本数据类型不熟悉。另一方面，也是把Leetcode过于高看，想得太复杂。通过第一道题目，对于Python的强大又有所体会，对于这一类算法的题目，使用Python来编程，编程语言和人的分析过程很接近，不需要耗费大量时间在自然语言和计算机程序的转换过程中。

Reverse String

题目简述

Write a function that takes a string as input and returns the string reversed.
Example: Given s = “hello”, return “olleh”.

Python 程序

Python中的字符串类型是没有reverse方法的，但是列表类型（list）有这个方法。所以首先使用list()函数来转换类型，将str转换成list，再通过方法reverse()实现list的反转，最后使用join方法来将字符串列表重新拼接成一个字符串。

总结

通过这道题目，主要学习了Python的list类型和str类型，以及相关的一些方法的使用。list()函数可以很方便的将传入的参数分解成一个列表；reverse()方法，可以将列表的元素倒转；字符串的join()方法可以拼接字符串。其他在这一程序中没有用到的方法，如list的append、remove、insert等也有所了解。

Nim Game

题目简述

You are playing the following Nim Game with your friend: There is a heap of stones on the table, each time one of you take turns to remove 1 to 3 stones. The one who removes the last stone will be the winner. You will take the first turn to remove the stones.
Both of you are very clever and have optimal strategies for the game. Write a function to determine whether you can win the game given the number of stones in the heap.
For example, if there are 4 stones in the heap, then you will never win the game: no matter 1, 2, or 3 stones you remove, the last stone will always be removed by your friend.

Python 程序

假设你和你的朋友两个人正在分一堆钻石，你们商量好两个人轮流取，每次取得钻石数量为1~3颗，谁拿到了最后一颗，谁就赢了。虽然不知道为何不是以谁拿的数量多为标准，但是假设就是这么无聊吧。
假设现在是到你取钻石的时候，如果钻石数量在1~3，那么你赢了。如果数量为4，那么不管你取多少，都不可能会赢。如果数量在5，你取走一颗，留下4颗，那么不管对手取多少，你都会获胜。如果取走2或者3颗，对手将会获胜。所以，最后一轮取之前，谁能够给对手留下4枚，谁就能获胜。继续分析，如果为6或7颗，留下4颗，那么你最终将获胜。如果数量为8颗，在双方都足够聪明的情况下，对方会获胜……分析可以发现，当总数量为4的倍数时，对方将会获胜，而不为4的倍数时，第一个拿钻石的人将会获胜。

总结

虽然这一个题目乍看之下，觉得很复杂，需要分析双方的策略，但是通过分析归纳得到规律，就可以转换成简单的解法。本解法的正确性关键在于一个前提：参与双方都足够聪明。通过这个题目，我体会到表面上看起来复杂的东西，往往其本质很简单，而有些看起来简单的往往又比较复杂。
